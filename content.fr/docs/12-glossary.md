# Glossaire

* ACK: Acknowledgment (Accusé de réception)
* AVP: Audio and Video profile (Profil audio et vidéo)
* B-Frame: [Bi-directional Predicted Frame](../06-media-communication/#intra-and-inter-frame-compression) (Image prédite bidirectionnelle). Une image partielle, est une modification des images précédentes et futures.
* DCEP: [Data Channel Establishment Protocol](../07-data-communication/#dcep) (Protocole d'établissement de canal de données) défini dans la [RFC 8832](https://datatracker.ietf.org/doc/html/rfc8832)
* DeMux: Demultiplexer (Démultiplexeur)
* DLSR: Delay since last sender report (Délai depuis le dernier rapport d'expéditeur)
* DTLS: [Datagram Transport Layer Security](../04-securing/#dtls) (Sécurité de la couche de transport de datagramme) défini dans la [RFC 6347](https://datatracker.ietf.org/doc/html/rfc6347)
* E2E: End-to-End (De bout en bout)
* FEC: [Forward Error Correction](../06-media-communication/#forward-error-correction) (Correction d'erreur directe)
* FIR: [Full INTRA-frame Request](../06-media-communication/#full-intra-frame-request-fir-and-picture-loss-indication-pli) (Demande d'image INTRA complète)
* G.711: Un codec audio à bande étroite
* GCC: [Google Congestion Control](../06-media-communication/#google-congestion-control-gcc) (Contrôle de congestion Google) défini dans le [draft-ietf-rmcat-gcc-02](https://datatracker.ietf.org/doc/html/draft-ietf-rmcat-gcc-02)
* H.264: Codage vidéo avancé pour les services audiovisuels génériques
* H.265: Spécification de conformité pour le codage vidéo haute efficacité ITU-T H.265
* HEVC: High Efficiency Video Coding (Codage vidéo haute efficacité)
* HTTP: Hypertext Transfer Protocol (Protocole de transfert hypertexte)
* HTTPS: HTTP Over TLS, défini dans la [RFC 2818](https://datatracker.ietf.org/doc/html/rfc2818)
* I-Frame: [Intra-coded Frame](../06-media-communication/#intra-and-inter-frame-compression) (Image intra-codée). Une image complète, peut être décodée sans rien d'autre.
* ICE: [Interactive Connectivity Establishment](../03-connecting/#ice) (Établissement de connectivité interactive) défini dans la [RFC 8445](https://datatracker.ietf.org/doc/html/rfc8445)
* INIT: Initiate (Initier)
* IoT: Internet of Things (Internet des objets)
* IPv4: Internet Protocol, Version 4 (Protocole Internet, version 4)
* IPv6: Internet Protocol, Version 6 (Protocole Internet, version 6)
* ITU-T: International Telecommunication Union Telecommunication Standardization Sector (Secteur de normalisation des télécommunications de l'Union internationale des télécommunications)
* JSEP: [JavaScript Session Establishment Protocol](../02-signaling/#what-is-the-session-description-protocol-sdp) (Protocole d'établissement de session JavaScript) défini dans la [RFC 8829](https://datatracker.ietf.org/doc/html/rfc8829)
* MCU: [Multi-point Conferencing Unit](../08-applied-webrtc/#mcu) (Unité de conférence multipoint)
* mDNS: [Multicast DNS](../03-connecting/#mdns) défini dans la [RFC 6762](https://datatracker.ietf.org/doc/html/rfc6762)
* MITM: Man-In-The-Middle (Homme au milieu)
* MTU: Maximum Transmission Unit (Unité de transmission maximale), la taille du paquet
* MUX: Multiplexing (Multiplexage)
* NACK: Negative Acknowledgment (Accusé de réception négatif)
* NADA: [network-assisted dynamic adaptation](../06-media-communication/#bandwidth-estimation-alternatives) (adaptation dynamique assistée par réseau) défini dans le [draft-zhu-rmcat-nada-04](https://tools.ietf.org/html/draft-zhu-rmcat-nada-04)
* NAT: [Network Address Translation](../03-connecting/#nat-mapping) (Traduction d'adresse réseau) défini dans la [RFC 4787](https://datatracker.ietf.org/doc/html/rfc4787)
* Opus: Un codec audio totalement ouvert, libre de droits et très polyvalent
* P-Frame: [Predicted Frame](../06-media-communication/#intra-and-inter-frame-compression) (Image prédite). Une image partielle, contenant uniquement les changements par rapport à l'image précédente.
* P2P: Peer-to-Peer (Pair-à-pair)
* PLI: [Picture Loss Indication](../06-media-communication/#full-intra-frame-request-fir-and-picture-loss-indication-pli) (Indication de perte d'image)
* PPID: [Payload Protocol Identifier](../07-data-communication/#payload-protocol-identifier) (Identifiant de protocole de charge utile)
* REMB: [Receiver Estimated Maximum Bitrate](../06-media-communication/#tmmbr-tmmbn-and-remb) (Débit binaire maximum estimé par le récepteur)
* RFC: Request for Comments (Demande de commentaires)
* RMCAT: [RTP Media Congestion Avoidance Techniques](../06-media-communication/#generating-a-bandwidth-estimate) (Techniques d'évitement de congestion média RTP)
* RR: Receiver Report (Rapport du récepteur)
* RTCP: [RTP Control Protocol](../10-history-of-webrtc/#rtp) (Protocole de contrôle RTP) défini dans la [RFC 3550](https://datatracker.ietf.org/doc/html/rfc3550)
* RTP: [Real-time transport protocol](../10-history-of-webrtc/#rtp) (Protocole de transport en temps réel) défini dans la [RFC 3550](https://datatracker.ietf.org/doc/html/rfc3550)
* RTT: Round-Trip Time (Temps aller-retour)
* SACK: Selective Acknowledgment (Accusé de réception sélectif)
* SCReAM: [Self-Clocked Rate Adaptation for Multimedia](../06-media-communication/#bandwidth-estimation-alternatives) (Adaptation de débit auto-synchronisée pour multimédia) défini dans le [draft-johansson-rmcat-scream-cc-05](https://tools.ietf.org/html/draft-johansson-rmcat-scream-cc-05)
* SCTP: [Stream Control Transmission Protocol](../07-data-communication/#stream-control-transmission-protocol) (Protocole de transmission de contrôle de flux) défini dans la [RFC 4960](https://datatracker.ietf.org/doc/html/rfc4960)
* SDP: [Session Description Protocol](../02-signaling/#what-is-the-session-description-protocol-sdp) (Protocole de description de session) défini dans la [RFC 8866](https://datatracker.ietf.org/doc/html/rfc8866)
* SFU: [Selective Forwarding Unit](../08-applied-webrtc/#selective-forwarding-unit) (Unité de transfert sélectif)
* SR: Sender Report (Rapport de l'expéditeur)
* SRTP: [Secure Real-time Transport Protocol](../04-securing/#srtp) (Protocole de transport en temps réel sécurisé) défini dans la [RFC 3711](https://datatracker.ietf.org/doc/html/rfc3711)
* SSRC: Synchronization Source (Source de synchronisation)
* STUN: [Session Traversal Utilities for NAT](../03-connecting/#stun) (Utilitaires de traversée de session pour NAT) défini dans la [RFC 8489](https://datatracker.ietf.org/doc/html/rfc8489)
* TCP: Transmission Control Protocol (Protocole de contrôle de transmission)
* TLS: The Transport Layer Security (Sécurité de la couche de transport) défini dans la [RFC 8446](https://datatracker.ietf.org/doc/html/rfc8446)
* TMMBN: [Temporary Maximum Media Stream Bit Rate Notification](../06-media-communication/#tmmbr-tmmbn-and-remb) (Notification de débit binaire de flux média maximum temporaire)
* TMMBR: [Temporary Maximum Media Stream Bit Rate Request](../06-media-communication/#tmmbr-tmmbn-and-remb) (Demande de débit binaire de flux média maximum temporaire)
* TSN: [Transmission Sequence Number](../07-data-communication/#transmission-sequence-number) (Numéro de séquence de transmission)
* TURN: [Traversal Using Relays around NAT](../03-connecting/#turn) (Traversée utilisant des relais autour de NAT) défini dans la [RFC 8656](https://datatracker.ietf.org/doc/html/rfc8656)
* TWCC: [Transport Wide Congestion Control](../06-media-communication/#transport-wide-congestion-control) (Contrôle de congestion large transport)
* UDP: User Datagram Protocol (Protocole de datagramme utilisateur)
* VP8, VP9: Technologies de compression vidéo hautement efficaces ("codecs" vidéo) développées par le WebM Project. Tout le monde peut utiliser ces codecs sans frais de licence.
* WebM: Un format de fichier multimédia ouvert conçu pour le web.
* WebRTC: Web Real-Time Communications (Communications en temps réel sur le web). [W3C WebRTC 1.0: Real-Time Communication Between Browsers](https://www.w3.org/TR/webrtc/)
